Date: 2026-05-02
Backend Base URL: http://localhost:5000
Frontend Base URL: http://localhost:5173

============================================================
0) BEFORE STARTING
============================================================
IMPORTANT: Error Handling Update (April 2026)
- All error responses are now displayed via ErrorNotification component
- Plain text alerts have been replaced with toast-style notifications  
- Error UI shows red left border with AlertCircle icon
- Errors auto-dismiss after 5 seconds or can be manually closed
- Backend should return proper error codes: 400, 401, 403, 404, 500

Updated pages with ErrorNotification:
✓ AdminHomePage, Checkout, DriverDeliveryApp, OrderTracking
✓ CreateExchange, ExchangeDetail, ExchangeMarketplace
✓ VendorProducts, ExchangeChat
Form validation errors remain inline for better UX context.

1. Start backend:
   npm run dev --prefix backend

2. Start frontend (optional for UI testing):
   npm run dev --prefix frontend

3. Create Postman Environment variables:
   baseUrl = http://localhost:5000
   frontendUrl = http://localhost:5173
   token = 
   customerUserId = 
   vendorUserId = 
   adminUserId = 
   productId = 
   categoryId = 
   sellerId = 
   sessionId = 
   listingId = 
   matchId = 
   driverId = 
   deliveryId = 
   orderId = 
   reviewId = 
   languageCode = hi

4. Common header for protected routes:
   Authorization: Bearer {{token}}

5. Error Response Guidelines:
   - 400: Bad Request (validation failed, missing fields)
   - 401: Unauthorized (invalid/missing token)
   - 403: Forbidden (insufficient permissions, user doesn't own resource)
   - 404: Not Found (resource doesn't exist)
   - 500: Server Error (backend issue)
   - Error messages now show in ErrorNotification component on frontend

6. Test style for every endpoint:
   A) Valid request (expected success: 200/201)
   B) Invalid request/body (expected: 400/401/403/404)
   C) For protected routes: test missing/malformed token (expected: 401/403)
   D) For user-specific routes: test with different user id (expected: 403)

15) HOW TO CHECK RESULT PROPERLY IN POSTMAN
============================================================
For each request check:
1. Status code
2. Response body keys
3. For create/update/delete, verify by calling GET endpoint after it
4. Save IDs/tokens in environment for next requests

Helpful Postman Tests tab snippets:

A) Status check
pm.test("Status is 200 or 201", function () {
  pm.expect([200, 201]).to.include(pm.response.code);
});

B) Save token from login
pm.test("Save token", function () {
  const body = pm.response.json();
  if (body.token) pm.environment.set("token", body.token);
  if (body.user && body.user.id) pm.environment.set("customerUserId", body.user.id);
});

C) Save product id from products list
pm.test("Save first product id", function () {
  const body = pm.response.json();
  if (Array.isArray(body) && body.length > 0 && body[0].id) {
    pm.environment.set("productId", body[0].id);
  }
});
