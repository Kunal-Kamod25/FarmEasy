FarmEasy API One-by-One Testing Guide (Postman / Thunder Client)
Date: 2026-03-24
Backend Base URL: http://localhost:5000

============================================================
0) BEFORE STARTING
============================================================
1. Start backend:
   npm run dev --prefix backend

2. Create Postman Environment variables:
   baseUrl = http://localhost:5000
   token =
   customerUserId =
   vendorUserId =
   adminUserId =
   productId =
   categoryId =
   sellerId =
   sessionId =

3. Common header for protected routes:
   Authorization: Bearer {{token}}

4. Test style for every endpoint:
   A) Valid request (expected success)
   B) Invalid request/body (expected 400/401/403/404)
   C) For protected routes: test missing token (expected 401/403)

14) QUICK FAILURE TESTS YOU MUST RUN
============================================================
1. Protected endpoint without token -> expect 401/403
2. Protected endpoint with malformed token -> expect 403
3. Profile/orders endpoint with other user id -> expect 403
4. Cart add without productId -> expect 400
5. Login without identifier/password -> expect 400
6. Create product review without rating -> expect 400
7. Create vendor review without verified purchase -> expect 400
8. Create duplicate review for same product -> expect 400
9. Update/delete review not owned by user -> expect 403
10. Create review with invalid rating (0 or 6) -> expect 400

============================================================
15) HOW TO CHECK RESULT PROPERLY IN POSTMAN
============================================================
For each request check:
1. Status code
2. Response body keys
3. For create/update/delete, verify by calling GET endpoint after it
4. Save IDs/tokens in environment for next requests

Helpful Postman Tests tab snippets:

A) Status check
pm.test("Status is 200 or 201", function () {
  pm.expect([200, 201]).to.include(pm.response.code);
});

B) Save token from login
pm.test("Save token", function () {
  const body = pm.response.json();
  if (body.token) pm.environment.set("token", body.token);
  if (body.user && body.user.id) pm.environment.set("customerUserId", body.user.id);
});

C) Save product id from products list
pm.test("Save first product id", function () {
  const body = pm.response.json();
  if (Array.isArray(body) && body.length > 0 && body[0].id) {
    pm.environment.set("productId", body[0].id);
  }
});

Auth -> Products -> Cart -> Orders -> Vendor -> Reviews -> Admin.

This is the fastest way to learn API testing deeply.
